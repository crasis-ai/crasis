## Notes

- **ONNX sizes are post int8 quantization** — roughly 4x smaller than the float32 base model footprint.
  BERT-Tiny (17 MB float32) → ~4.3 MB quantized. BERT-Mini (43 MB float32) → ~10.8 MB quantized.
- **Latency** is median over 50 single-inference runs on CPU (single thread), no batching.
- **email-urgency** is 3-class (critical / high / low) trained on BERT-Mini. 95.36% synthetic accuracy. Holdout: 60.00% accuracy, F1 0.5631 (35.4pp gap). Critical class has the best recall (0.90) but high class recall collapses to 0.20 — the model struggles to distinguish high from critical/low.
- **spam-filter** was rebuilt on BERT-Tiny; ONNX size dropped from 10.86 MB to 4.29 MB and latency from 2.67 ms to 0.56 ms. Holdout: 80.00% accuracy, F1 0.7947 (18.9pp gap).
- **support-router** is a 5-class multiclass trained on BERT-Tiny — tighter size budget, lower natural accuracy ceiling than email-urgency's BERT-Mini. 96.25% exceeds the 90% gate.
- **Training times** are on RTX 4060 (8 GB), 10 epochs, batch size 32, fp16.
- Data generation for binary tasks uses `deepseek/deepseek-chat-v3.1` (~20 min / 3,000 samples, 4 workers).
  Multiclass uses `meta-llama/llama-3.3-70b-instruct`.

## Accuracy Analysis

All accuracy figures are synthetic eval results. Initial holdout evaluation on hand-authored examples shows real-world accuracy lower for each model. Treat published figures as training convergence metrics, not field-validated performance claims.

**Synthetic accuracy** (the Eval column from training) reflects performance within the LLM-generated data distribution — not real-world text. All 10 specialists hit synthetic gates of 85–99%.

**Holdout accuracy** is measured by running the deployed ONNX model against hand-authored examples in `tests/fixtures/<name>.jsonl`. These examples were written without any generator templates, covering edge cases and phrasing patterns the synthetic pipeline tends to flatten out.

**The gap is the signal.** Large gaps indicate the model learned the generator's linguistic fingerprints more than the underlying task. Two specialists have small gaps (≤3pp) and can be considered field-ready. The rest have meaningful gaps that warrant more training data diversity, a hybrid strategy, or a larger base model.

| Gap range | Specialists | Interpretation | Fix |
|---|---|---|---|
| <5pp | whatsapp-triage, invoice-intent | Synthetic data transfers well; task boundary is crisp | None needed |
| 5–10pp | availability-handler, pricing-detector | Mild distribution shift; monitor in production | Optional: `crasis mix` with real examples |
| 10–20pp | support-router, social-classifier, meeting-parser, spam-filter, sentiment-gate | Meaningful gap; consider hybrid training data | `crasis mix --real-data ./your-examples.jsonl` |
| >20pp | email-urgency (35.4pp) | Severe distribution shift; high class recall collapses to 0.20; recommend hybrid strategy or real-world data | `crasis mix` with 200–300 real `high`-class examples — see Findings below |

---

## Findings

**email-urgency high-class collapse:** The model detects `critical` (recall 0.90) and `low` (recall 0.70) but nearly misses `high` (recall 0.20, precision 0.50). The synthetic generator likely conflates `high` and `critical` phrasing — both urgency levels use escalatory language, so the model learns a two-pole distribution (critical vs. low) with no stable decision boundary for `high`. The fix is real labeled examples that draw a clear line between "this needs attention today" and "this needs attention right now." A hybrid strategy with even 200–300 real `high`-class examples would likely resolve the collapse.

To reproduce any holdout result:

```bash
crasis eval -s specialists/<name>/spec.yaml \
    -m ./models/<name>-onnx \
    --holdout tests/fixtures/<name>.jsonl
```
